Devoxx API

Welcome to Devoxx France REST API.

API is in Beta mode

Please, report issue with our bug tracker

Please, use a JSON client and discover our very simple REST API.

The User-Agent HTTP field is mandatory.

Each Response contains:

  • an ETag. Please, check and use If-None-Match to avoid too many calls to the server.
  • a Links HTTP Header (RFC 6906) with a short documentation about each resource.

Resources

The API serves the following content:

  • A collection of Conferences
  • A conference
  • A list of speakers
  • A speaker
  • A talk
  • A set of schedules
  • A schedule
  • A link

See all conferences

curl -H "Accept: application/json" @routes.RestAPI.showAllConferences().absoluteURL()

See details for an event

curl -H "Accept: application/json" @routes.RestAPI.showConference(Conference.devoxxFrance2014.eventCode).absoluteURL()

See all speakers for this event

curl -H "Accept: application/json" @routes.RestAPI.showSpeakers(Conference.devoxxFrance2014.eventCode).absoluteURL()

See details for a specific speaker

curl -H "Accept: application/json" @routes.RestAPI.showSpeaker(Conference.devoxxFrance2014.eventCode, "25b6eeb75c18e3465d5cddf2be297b8863006551").absoluteURL()

See details for a talk

curl -H "Accept: application/json" @routes.RestAPI.showTalk(Conference.devoxxFrance2014.eventCode, "QDI-693").absoluteURL()

See the list of Schedules

curl -H "Accept: application/json" @routes.RestAPI.showAllSchedules(Conference.devoxxFrance2014.eventCode).absoluteURL()

See the schedule for Wednesday

curl -H "Accept: application/json" @routes.RestAPI.showScheduleFor(Conference.devoxxFrance2014.eventCode,"wednesday").absoluteURL()

French is also supported, use Accept-language header :

curl -H "Accept: application/json" -H "Accept-language: fr-FR" @routes.RestAPI.showSpeaker(Conference.devoxxFrance2014.eventCode, "25b6eeb75c18e3465d5cddf2be297b8863006551").absoluteURL()
}
